# Как редактировать контент

После миграции проект использует JSON как основной источник данных. Компоненты интерфейса больше не нужно править для обычных контентных изменений.

## Где что хранится

### 1. Сценарии, ответы и профили результата

Файл: [simulator-content.json](/C:/Users/evstygney/Documents/Контент-план/psyvit-soft-skills-simulator/data/simulator-content.json)

Здесь хранятся:

- все сцены симулятора;
- контекст и реплика второй стороны;
- варианты ответов;
- тип каждого ответа;
- вес каждого ответа;
- психологический комментарий;
- профили результата.

### 2. Тексты экранов и ссылки

Файл: [site-content.json](/C:/Users/evstygney/Documents/Контент-план/psyvit-soft-skills-simulator/data/site-content.json)

Здесь хранятся:

- метаданные сайта;
- брендовые подписи;
- welcome-экран;
- тексты результата;
- страница программы;
- подписи кнопок;
- ссылки переходов.

## Как править напрямую

Подходит, если правки делает технический человек или кто-то аккуратно работает с JSON.

### В `simulator-content.json`

Структура верхнего уровня:

- `scenarios`
- `styleProfiles`

У каждой сцены:

- `id`
- `title`
- `context`
- `prompt`
- `options`

У каждого ответа:

- `id`
- `text`
- `style`
- `weight`
- `comment`

Допустимые значения `style`:

- `avoiding`
- `defensive`
- `mature`

### В `site-content.json`

Основные блоки:

- `metadata`
- `brand`
- `links`
- `welcome`
- `result`
- `course`
- `legacyCta`

Если нужно поменять переходы по кнопкам:

- главная ссылка симулятора: `links.home`
- ссылка на программу: `links.course`

## Как править через CSV

Подходит, если контент сначала смотрят редакторы.

Редакторский файл:
- [redakturnyy-paket-simulyatora.csv](/C:/Users/evstygney/Documents/Контент-план/psyvit-soft-skills-simulator/docs/redakturnyy-paket-simulyatora.csv)

После правок CSV можно импортировать обратно в проект:

```bash
npm run content:import-csv
```

Если файл лежит в другом месте:

```bash
npm run content:import-csv -- path/to/file.csv
```

### Что обновляет импорт из CSV

- сцены;
- ответы;
- типы и веса;
- комментарии;
- профили результата;
- заголовок и текст блока результата;
- тексты страницы программы;
- пункты программы;
- правый информационный блок;
- текст рекомендованного следующего шага.

### Что CSV сейчас не обновляет

Эти поля остаются в [site-content.json](/C:/Users/evstygney/Documents/Контент-план/psyvit-soft-skills-simulator/data/site-content.json) и правятся вручную:

- welcome-экран;
- брендовые подписи;
- метаданные сайта;
- ссылки `links.home` и `links.course`;
- часть служебных подписей кнопок и секций, которых нет в CSV.

## Как проверить после правки

Локальный запуск:

```bash
npm run dev
```

Если Next.js сломал dev-кэш:

```bash
npm run reset-dev
```

Проверка production-сборки:

```bash
npm run build
```

## Что не стоит делать

- не удалять `id` у сцен и ответов без причины;
- не менять `style` на произвольный текст;
- не превращать `weight` в строку или пустое значение;
- не удалять целые поля из JSON;
- не править интерфейсные компоненты, если меняется только контент.
